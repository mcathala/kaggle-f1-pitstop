## Goal

Maximise ROC-AUC on the Kaggle test set, mirrored by 5-fold OOF AUC. See [../docs/eda.md §8](../docs/eda.md#8-traintest-split-structure-important-for-cv) for why row-level StratifiedKFold is the correct CV.

## CV protocol (the bar all results are measured against)

- 5-fold `StratifiedKFold(shuffle=True, random_state=42)` on `Year × PitNextLap`.
- Test predictions = mean of fold models, each at its own `best_iteration`.
- AUC computed on out-of-fold predictions concatenated across folds.
- LightGBM params and seed fixed to baseline; only the experimental change moves.

If the CV protocol changes, that's its own experiment (and it invalidates comparison to prior numbers).

## Workflow per cycle

1. **REVIEW** — Read the leaderboard, last few experiment writeups, residuals/feature-importance on current best. Identify the top weakness.
2. **PROPOSE** — Pick ONE hypothesis. Open `experiments/NNN_<short>.md` with sections: hypothesis, expected impact, overfit risk, validation gates.
3. **IMPLEMENT** — New branch `exp/<short-desc>`. Keep CV folds + seed fixed.
4. **VALIDATE** — OOF AUC + per-fold std, train-val gap, per-year and per-compound AUC. Reject if val improves but train-val gap widens > 0.005 without LB confirmation.
5. **DOCUMENT** — Update the experiment file with results table + decision (Kept / Reverted / Pending). Update this leaderboard. Update `../CHANGELOG.md`.
6. **REPORT** — Summarise the result, then decide deliberately what to commit and whether to spend a submission slot.

## Hard rules

- Commit, push, and submit deliberately — never on a whim mid-experiment.
- CV folds + seed identical across experiments for comparability.
- Treat any change touching data-leakage risk, CV strategy, or submission logic as high-stakes — re-derive and double-check before trusting it.
